# 一句话自动开发（给用户的说明）

> 你**不用**学 PRD、YAML、Multi-Agent Prompt。  
> 在 Cursor 里**发一句话**，Agent 会自动规划、拆分、派多个子 Agent 写代码、联调验收。

---

## 你怎么用（两阶段）

**第一步 — 要计划：**

```
帮我规划开放平台 P0 开发，先出计划，我确认后再执行。
```

Agent 只出《开发计划》，**不改代码**。

**第二步 — 你点头：**

```
同意执行
```

Agent 才会派执行 Agent 写代码、Review Agent 验收。

---

## Agent 之间怎么「通知」？（你不需要参与）

**没有**单独的「通知系统」或「Agent 群聊」。实际是：

```text
你 ──对话──▶ 总编排 Agent（PM + 项目经理）
                    │
                    │ 你回复「同意执行」后
                    │
                    ├── Task 启动 ──▶ 执行 Agent A（例如后端）
                    ├── Task 启动 ──▶ 执行 Agent B（例如前端）  ← 同一 Wave 可并行
                    └── Task 启动 ──▶ 执行 Agent C（例如网关）
                              │
                              │ 各自干完，把结果文字回报给总编排
                              ▼
                    总编排 ──Task──▶ Review Agent（联调验收）
                              │
                              ▼
                    总编排把摘要发给你
```

- **「创建智能体」** = 总编排 Agent 在后台调 **Task 工具**，相当于临时雇一个只干一件事的工人；不是你在 Cursor 里新建聊天。
- **「通知其他 Agent」** = Task 的 **prompt 里写清任务**（改哪个目录、交付什么）；工人做完 **返回一段结果** 给总编排，总编排再决定下一 Wave。
- **子 Agent 互相不聊天**；对齐靠同一份落地方案 + 计划表里的边界。

---

## 角色对照

| 角色 | 谁 | 何时 |
|------|-----|------|
| PM | 总编排 Agent | 你先要计划时 |
| 执行 | Task 子 Agent × N | 你点头后 |
| Review | Task 子 Agent（Integration） | 执行 Wave 完成后 |
| 你 | 只审批计划、看最终摘要 | 发需求 + 说「同意执行」 |

---

## 你需要知道的限制（诚实说明）

| 期望 | 现实 |
|------|------|
| 完全零点击 | 你至少要**发第一条消息**启动 Agent |
| 7 个独立 Cursor 窗口自动跑 | **不需要** — 一个对话里 Agent 会派子 Agent（Task） |
| 永远不用看代码 | 大功能仍建议你看一眼 diff；Agent 可能漏边界 case |
| 自动上线生产 | 不会 — commit / PR / 部署需你明确说「提交/建 PR」 |
| 关电脑继续跑 | 需 Cursor **Automations**（云端自动化）；本地 Agent 会话关了就停 |

---

## 可选：云端持续自动化

若希望「定时/触发后自动继续开发」，可用 Cursor **Automations**（需登录 Cursor 账号）。  
在对话里说：`帮我创建一个 Automation，触发后执行开放平台 P0 自动开发` — Agent 可协助配置。
