> **目标**：按 [开放平台 API 接口规范](../external/开放平台API接口规范.md) 交付对外 API 能力；运营通过**独立前端子应用**完成 Partner 开通/凭证/能力/监控；后端在 **open-api-service** 扩展，公网流量经 **partner-gateway**。

> **相关文档**：[页面设计](./开放平台集成管理后台-页面设计.md) · [multi-agent 执行 Prompts](./multi-agent-执行Prompts.md) · [组件职责与接口映射](./组件职责与接口映射.md) · [联调手册 P0](./联调手册-P0.md)

## 一、背景与边界

### 1.1 要交付什么

| 交付物 | 说明 |
|--------|------|
| **对外 API** | `/api/open/v1/*` + `/oauth/token`，Partner 机机集成 |
| **网关** | partner-gateway：鉴权、能力码、限流、注入 `X-Partner-Id` |
| **统一管理服务** | open-api-service：Partner 身份、Token、业务编排、治理审计 |
| **集成管理后台** | 运营界面：登记 Partner、生成凭证、配置能力、（P1+）监控 |

规范 §2.1 明确：**Partner 由运营在管理后台登记**，第三方不能自注册。集成管理后台是整条链路的**运营入口**。

### 1.2 工程边界

| 工程 | 维护方 | 本方案中的角色 |
|------|--------|----------------|
| `asset-manage-master` | 业务前端团队 | 主应用：登录、顶栏菜单、qiankun 注册 |
| `clover-front`（`/system`） | 公共研发 | **不改代码**；仅可能协调菜单库加 1 条入口 |
| `asset-openplatform-manage`（新建） | 业务前端团队 | 集成管理后台子应用 |
| `open-api-service` | 业务后端团队 | 管理 API + 开放 API + 治理 |
| `partner-gateway` | 业务后端团队 | 公网 Partner 流量入口 |

**路径速查**：

| 用途 | 路径 |
|------|------|
| 主应用 | `project_frontend/asset/asset-manage-master/` |
| 子应用（新建） | `project_frontend/asset/asset-openplatform-manage/` |
| 子应用 UI 参考 | `project_frontend/asset/asset-other-manage/src/views/system/userManage/UserInfoList.vue` |
| qiankun 模板 | `project_frontend/asset/asset-newleak-manage/` |
| 后端服务 | `project_backend/svmp/open-api-service/` |
| 网关 | `project_backend/svmp/partner-gateway/` |
| OpenAPI | `svmp/openapi/v1/openapi.yaml` |

### 2.1 两类请求，两套鉴权

| 请求类型 | 路径示例 | 鉴权 | axios 实例 |
|----------|----------|------|------------|
| 门户 | `/clover/v3/*` | `Authorization: JWT`（主应用同步） | 可选复用主应用 token |
| Partner 管理 | `/internal/admin/partners` | `X-Internal-Admin-Key` | **`openApiRequest.js`（独立）** |

子应用 mount 时仍从主应用同步 JWT（与 `asset-newleak-manage` 相同），但 **Partner 管理 API 不使用 JWT**，只用 Admin Key。

### 3.4 环境变量

| 变量 | 说明 | 存放 |
|------|------|------|
| `VUE_APP_OPEN_API_BASE_URL` | open-api-service 地址 | `.env.development` / 部署 conf |
| `VUE_APP_OPEN_API_ADMIN_KEY` | 与 `open-api.admin.api-key` 一致 | **`.env.development.local`（gitignore）** |
| `VUE_APP_OPEN_API_PUBLIC_BASE` | 对外域名（接入说明用） | 如 `https://vuln.example.com` |

### 3.5 openApiRequest.js 行为

1. `baseURL` = `VUE_APP_OPEN_API_BASE_URL`
2. 请求头：`X-Internal-Admin-Key: process.env.VUE_APP_OPEN_API_ADMIN_KEY`
3. 启动检测：Key 为空 → 全局 `adminKeyMissing = true`，页面顶栏 `a-alert type="error"`
4. 响应：`{ code, message, data, requestId }`，`code !== 0` → `notification.error(message)`
5. HTTP 401/403 → 「Admin Key 无效或未配置，请联系运维」

## 四、主应用集成（asset-manage-master）

### 4.1 qiankun 注册

在 `src/main.js` 的 `registerMicroApps` 追加：

```javascript
{
  name: 'openPlatform',
  entry: MICRO_APP_OPENPLATFORM || isUseConfJsForString('VUE_APP_API_OPENPLATFORM_URL'),
  container: '#subapp-viewport2',
  activeRule: '/openPlatform',
  props: { data: { store, router } }
}
```

### 4.2 运行时配置

`public/conf/index.js` 追加：

```javascript
VUE_APP_API_OPENPLATFORM_URL: loc + ':13021/',
```

Docker/生产：`entrypoint.sh` 或 body attribute 注入内网 entry URL。

### 4.3 菜单（顶栏入口）

菜单数据来自 **`GET /clover/v3/pub/menu`**（clover 后端菜单库），**不是** clover-front 代码。

需在菜单库新增（协调公共研发或自有菜单管理权限）：

```text
开放平台（一级或挂在「系统管理」下）
└── 合作方管理    path: /openPlatform/partner
    └── （可选）调用记录  path: /openPlatform/invocation   [P1]
```

**子应用内部侧边栏**由 `router.config.js` 的 `asyncRouterMap` 静态维护（同 `asset-newleak-manage`），不依赖 clover-front。

开发期可直接访问：`http://localhost:13001/openPlatform/partner`。

### 6.3 执行平面 + 治理平面（P0–P3）

| 阶段 | 执行平面 | 治理平面 | 管理后台 |
|------|----------|----------|----------|
| **P0** | tasks CRUD、Partner 隔离、Token | `api_invocation` 表、requestId 全链路 | Partner/凭证/能力（本子应用） |
| **P1** | 实例 lifecycle、Webhook | Partner/API 成功率、Webhook 投递状态 | 调用记录页、统计 Tab |
| **P2** | exports | 配额/限流报表 | 运营看板 |
| **P3** | — | API 版本/弃用、告警、SLA | 版本/告警配置页 |

### 6.4 partner-gateway 关系

- 公网：`Partner → gateway → open-api-service`
- **`/internal/admin/**` 不得经 gateway 暴露**（已实现 40101）
- 集成管理后台 **直连内网** open-api-service
- Token：gateway 读 Redis `partner:token:{sha256}`，不签发 Token
