# 开放平台集成管理后台 · 页面设计

> **定位**：平台**运营/集成管理员**使用的 Web 界面，用于 Partner 登记、凭证交付、能力配置与（P1+）调用治理观测。  
> **不负责**：第三方 Partner 的业务操作界面（Partner 通过 API 集成，见 [external 规范](../external/开放平台API接口规范.md)）。  
> **相关文档**：[完整落地方案](./开放平台集成管理-完整落地方案.md) · [组件职责与接口映射](./组件职责与接口映射.md) · [API 治理与调用生命周期](./开放平台API治理与调用生命周期.md) · [multi-agent 执行 Prompts](./multi-agent-执行Prompts.md)

---

## 1. 两类「页面」不要混淆

| 类型 | 谁用 | 是否需要做 Web 页 |
|------|------|-------------------|
| **集成管理后台**（本文） | 平台运营、安全管理员 | **需要** — Partner 开通、凭证、能力、监控 |
| **Partner 业务系统** | 第三方 SIEM/SOC/ITSM | **不需要** — 调 `/api/open/v1/*`，无门户 |
| **Partner 开发者门户**（可选 P2+） | 第三方开发者自助查文档/密钥 | 可选，非 P0 |

## 2. 技术选型建议

| 项 | 建议 | 说明 |
|----|------|------|
| 前端工程 | **`asset-openplatform-manage` 独立子应用** | 见 [完整落地方案 §三](./开放平台集成管理-完整落地方案.md)；由 asset-manage-master qiankun 挂载 |
| 登录 | **复用主应用门户 JWT** | qiankun mount 同步 token；Partner 管理 API 仍用 Admin Key |
| 业务 API | **直连 open-api-service** | `/internal/admin/*`，请求头 `X-Internal-Admin-Key` |
| 菜单挂载 | clover 菜单库增加 `/openPlatform/partner` | 顶栏入口；子应用内路由自维护 |

> **说明**：clover-front 由公共研发维护，本方案**不扩展 clover-front**；UI 风格参考 `asset-other-manage` 用户管理列表。

### 2.1 前端调用链

```text
浏览器（运营人员）
  → 已登录 asset-manage-master（JWT，clover 门户）
  → qiankun 加载 asset-openplatform-manage（/openPlatform）
  → openApiRequest 调 open-api-service（内网 + X-Internal-Admin-Key）
       GET/POST /internal/admin/partners/...
```

| 请求 | 鉴权 |
|------|------|
| 主应用登录、顶栏菜单 | 门户 `Authorization: JWT` |
| Partner 管理 API | `X-Internal-Admin-Key`（环境变量配置，勿提交仓库） |

> open-api-service 的 `InternalAdminAuthFilter` 只校验 Admin Key，**不**使用 Partner Bearer Token。

### 2.2 环境变量（asset-openplatform-manage）

| 变量 | 示例 | 说明 |
|------|------|------|
| `VUE_APP_OPEN_API_BASE_URL` | `http://open-api-service:8080` | open-api-service 内网地址 |
| `VUE_APP_OPEN_API_ADMIN_KEY` | 部署时注入 | 与 `open-api.admin.api-key` 一致 |

开发期可在 `.env.development.local` 配置（加入 `.gitignore`）。

## 4. 页面清单与后端 API 映射

### 4.1 P0 必做页面

#### 页面 1：合作方列表 `PartnerList.vue`

| 项 | 说明 |
|----|------|
| 路由 | `/integration/partner` 或 `/system/openPlatform/partner` |
| API | `GET /internal/admin/partners?page=&size=` |
| 功能 | 表格分页、按 partnerId/名称搜索、状态标签、进入详情、新建 |

| 列名 | 字段 | 说明 |
|------|------|------|
| 接入方 ID | partnerId | 可点击进详情 |
| 名称 | partnerName | |
| 类型 | partnerType | 标签：SIEM / ITSM / SECOPS … |
| 状态 | status | ACTIVE / DISABLED |
| 限流 QPS | rateLimitQps | 空显示「不限」 |
| 能力数 | capabilities.length | 或摘要「任务读写」 |
| 操作 | — | 编辑、停用 |

**操作按钮**：「新建合作方」→ 跳转创建页或抽屉表单。

---

#### 页面 2：新建/编辑合作方 `PartnerForm.vue`

| 模式 | API |
|------|-----|
| 新建 | `POST /internal/admin/partners` |
| 编辑 | `PUT /internal/admin/partners/{partnerId}` |

**表单字段**（与 `CreatePartnerRequest` / `UpdatePartnerRequest` 对齐）：

| 字段 | 组件 | 必填 | 校验/说明 |
|------|------|:---:|----------|
| partnerId | Input | 新建✓ | 字母数字连字符，创建后**不可改** |
| partnerName | Input | ✓ | 如「SOC能力中心」 |
| partnerType | Select | ○ | SIEM / ITSM / ASSET / SECOPS / REGULATORY_PROXY / CUSTOM |
| status | Select | 编辑 | ACTIVE / DISABLED |
| capabilities | Checkbox.Group | ✓ | 见 §5 能力码中文名 |
| defaultCallbackUrl | Input | ○ | **Partner 自己的 Webhook 地址**，非平台域名 |
| rateLimitQps | InputNumber | ○ | ≥1；空=不限 |

**提交成功**：新建 → 跳转详情页；编辑 → 提示成功并刷新。

---

#### 页面 3：合作方详情 `PartnerDetail.vue`

| 项 | 说明 |
|----|------|
| 路由 | `/integration/partner/:partnerId` |
| API | `GET /internal/admin/partners/{partnerId}` |

**页头**：partnerName + partnerId + 状态 Tag +「编辑基本信息」

**Tab 1 · 基本信息**  
展示/编辑 partnerType、status、rateLimitQps、defaultCallbackUrl（可内联编辑或弹窗）。

**Tab 2 · 能力配置**  
Checkbox 展示已开通 capabilities，保存调用 `PUT` 全量更新。

**Tab 3 · 接入凭证**（重点）

| API | 用途 |
|-----|------|
| `GET .../credentials` | 凭证列表 |
| `POST .../credentials` | 生成新凭证 |

| 列名 | 字段 |
|------|------|
| 客户端 ID | clientId |
| 凭证 ID | credentialId |
| 状态 | status |
| 创建时间 | createdAt |

**「生成凭证」按钮** → Modal：

1. 调用 `POST .../credentials`
2. 展示 **clientId + clientSecret（仅一次）** + 复制按钮
3. 醒目提示：「请立即交付第三方，关闭后无法再次查看 secret」
4. 附带 **接入说明折叠面板**（可复制）：
   - Token URL：`POST {公网}/oauth/token`
   - API Base：`{公网}/api/open/v1`
   - grantType：`client_credentials`

**Tab 4 · 回调配置**  
展示 defaultCallbackUrl、webhook_secret 轮换（P1 API 未就绪则仅展示 URL）。

### 4.2 P1 页面（治理平面）

#### 页面 4：API 调用记录 `InvocationList.vue`

| API | `GET /internal/admin/invocations`（后端 P1 实现） |
|-----|--------------------------------------------------|
| 筛选 | partnerId、时间范围、operationId、response_code |
| 列 | requestId、partnerId、接口、code、耗时、时间 |

点击行 → **调用详情** `InvocationDetail.vue`：requestId、错误信息、关联 resourceId（taskId）。

#### 页面 5：合作方统计 `PartnerStats.vue`

可嵌入详情页 Tab 或独立看板：

| API | `GET /internal/admin/partners/{partnerId}/stats` |
|-----|--------------------------------------------------|
| 展示 | 今日调用量、成功率、Top 错误码、近 7 日折线图 |

## 6. 交互与体验要点

| 场景 | 设计 |
|------|------|
| 首次交付凭证 | Modal 强提醒 + 一键复制 JSON 包 |
| 停用 Partner | 二次确认；说明现有 Token 将失效（P1 revoke） |
| defaultCallbackUrl 填错 | 表单项说明：「平台向该地址 POST Webhook」 |
| API 错误 | 解析 open-api `ApiResponse`：`code !== 0` 用 `message` 提示 |
| 权限 | 菜单仅「集成管理员」角色可见（clover 菜单配置） |

## 7. 前端工程结构（clover-front 扩展）

```text
clover-front/src/
├── api/
│   └── openPlatform/
│       ├── partner.js          # list/get/create/update/credentials
│       └── invocation.js       # P1
├── views/
│   └── openPlatform/
│       ├── PartnerList.vue
│       ├── PartnerForm.vue
│       ├── PartnerDetail.vue
│       ├── components/
│       │   ├── CapabilityCheckboxGroup.vue
│       │   ├── CredentialCreateModal.vue
│       │   └── DeliveryGuidePanel.vue   # 接入说明可复制
│       └── InvocationList.vue           # P1
└── config/
    └── openPlatformCapabilities.js      # 能力码常量
```

**`api/openPlatform/partner.js` 示例约定**：

```javascript
import openApiRequest from '@/utils/openApiRequest'  // 独立 axios 实例 + Admin Key

export function listPartners (params) {
  return openApiRequest({ url: '/internal/admin/partners', method: 'get', params })
}
```

**`utils/openApiRequest.js`**：独立实例，不污染门户 JWT 逻辑；统一解析 `{ code, data, message }`。

## 12. 验收标准（P0 页面）

- [ ] 运营可在界面创建 Partner（含 capabilities、partnerType）
- [ ] 可生成凭证并复制 clientId/secret + 接入说明
- [ ] 列表与详情数据与 `GET /internal/admin/partners` 一致
- [ ] 未配置 Admin Key 时有明确错误提示
- [ ] 页面风格与 clover-front 用户管理列表一致（Ant Design 表格/表单）
