## 1. 总体链路

```text
Partner
  │  HTTPS
  ▼
morningglory ──路由──► open-api-service ──Feign/REST──► SVMP 执行层
  │  Partner 校验              │  业务 / 幂等 / Webhook
  │  capability / 限流         │
  │                            └── Feign ──► clover（Partner 配置 / 名称解析）
  │
  └── Token 校验 ◄── Redis / clover introspect

clover ◄── 集成管理后台（Partner CRUD、凭证）
  └── POST /oauth/token ◄── Partner（经 morningglory 路由）
```

**强制规则**

| 规则 | 说明 |
|------|------|
| Token 签发 | 仅 **clover**；morningglory **不签发** |
| Partner 上下文 | 仅信任 morningglory 注入的 **`X-Partner-Id`** |
| SVMP 调用 | 仅 **open-api-service**；Partner **不直连** |
| extTaskId | **不下发** SVMP；唯一约束 `(partner_id, ext_task_id)` |

## 2. morningglory · 集成网关

**职责**：TLS、路由、Partner Token 校验、capability 拦截、按 Partner 限流、向下游注入上下文。  
**不负责**：Token 签发、开放平台业务、SVMP 调用、Webhook 出站。

### 2.1 路由（Nacos 配置 · 🆕）

| 序号 | 路径模式 | 目标服务 | 说明 |
|------|----------|----------|------|
| R1 | `/api/open/v1/**` | `open-api-service` | 开放平台 REST（需 Partner 鉴权） |
| R2 | `/oauth/token` 或 `/api/open/v1/oauth/token` | `clover` | Partner 取 Token（**免** Partner 鉴权，见白名单） |
| R3 | `/internal/**` | 各内部服务 | 集群内 / 管理网段；**禁止**公网暴露 |

> 现有 discovery locator（`/{serviceId}/**`）继续服务门户流量；开放平台路径需**显式**路由，避免与 `/v3/pub/*` 混用。

### 2.2 过滤器逻辑（🔧 新增 `PartnerOpenApiFilter`）

| 步骤 | 编号 | 动作 | 失败响应 |
|------|------|------|----------|
| 路径分流 | G1 | 仅 `/api/open/v1/**` 启用 Partner 逻辑；其余走现有 `JWTAuthFilter` | — |
| Token 校验 | G2 | Redis `partner:token:{hash}` 或 Feign `clover` `/internal/token/introspect` | HTTP 200, `code=40101` |
| 主体类型 | G3 | `subjectType != PARTNER` → 拒绝 | `40101` |
| 能力匹配 | G4–G5 | 查「方法 + 路径 → capability」表 | `40301` |
| 上下文注入 | G6 | 设置 `X-Partner-Id`、`X-Request-Id`（可选 `X-Capabilities`） | — |
| 限流 | G7 | 按 `partnerId` QPS（Partner 表 `rate_limit_qps` 或 Sentinel） | `42901` |
| HMAC（P2） | G8 | `X-Api-Key` + `X-Signature` + `X-Timestamp` 校验 | `40101` |
| IP / mTLS（P3） | G9 | 连接层策略 | `40101` |

**建议过滤器顺序**：`IgnoreURL` → `PartnerOpenApiFilter`（G1–G9）→ `JWTAuthFilter`（门户）→ 下游。

### 2.3 路径 → capability 映射（G4–G5）

Base Path：`/api/open/v1`。与 [OpenAPI](../../openapi/v1/openapi.yaml) 及 [Partner 鉴权 §4.2](./开放平台Partner鉴权与隔离-落地方案.md) 一致。

| 方法 | 路径 | capability | 阶段 |
|------|------|------------|------|
| POST | `/tasks` | `TASK_WRITE` | P0 |
| GET | `/tasks` | `TASK_READ` | P0 |
| GET | `/tasks/{taskId}` | `TASK_READ` | P0 |
| POST | `/instances/search` | `INSTANCE_READ` | P0 |
| GET | `/instances/{vulInfoID}` | `INSTANCE_READ` | P0 |
| POST | `/instances/{vulInfoID}/verify` | `INSTANCE_VERIFY` | P1 |
| POST | `/instances/verify:batch` | `INSTANCE_VERIFY` | P1 |
| POST | `/instances/{vulInfoID}/remediate` | `INSTANCE_REMEDIATE` | P1 |
| POST | `/instances/{vulInfoID}/archive` | `INSTANCE_ARCHIVE` | P1 |
| POST | `/instances/{vulInfoID}/verify-fix` | `INSTANCE_VERIFY_FIX` | P1 |
| POST | `/instances/verify-fix:batch` | `INSTANCE_VERIFY_FIX` | P1 |
| GET | `/exports/{exportId}` | `EXPORT_READ` | P2 |
| GET | `/exports/{exportId}/download` | `EXPORT_READ` | P2 |
| GET | `/tasks/{taskId}/exports` | `EXPORT_READ` | P2 |
| POST | `/instances/{vulInfoID}/unfixable-records` | `INSTANCE_ARCHIVE` | P1（废弃别名，转发 archive） |

### 2.4 向下游注入请求头（G6）

| Header | 来源 | 说明 |
|--------|------|------|
| `X-Partner-Id` | Token 上下文 | open-api-service **唯一**可信 Partner 标识 |
| `X-Request-Id` | 网关生成或透传 | 全链路追踪 |
| `Authorization` | 透传或剥离 | 建议剥离，避免业务层误用 Token 解析 partnerId |

### 2.5 白名单配置（🔧）

| 路径 | 是否免 Partner 鉴权 | 说明 |
|------|---------------------|------|
| `/oauth/token` | ✓ | Partner 取 Token |
| `/api/open/v1/oauth/token` | ✓ | 同上（若采用统一前缀） |
| `/v3/pub/login` 等门户路径 | ✓（现有） | **不得**包含 `/api/open/v1/**` |
| `/api/open/v1/**` | ✗ | 必须 Partner Token |

### 2.6 复用现有能力

| 能力 | 现状 | 开放平台用法 |
|------|------|--------------|
| `JWTAuthFilter` | ✅ | 门户 JWT / CLIENT / Bearer |
| Redis | ✅ | 共用连接池；Partner 键前缀独立 |
| Sentinel | 🔧 依赖已有，默认关闭 | 按 `partnerId` 限流 |
| `BurpSuiteSimpleProtectFilter` | ✅ 可选 | 与开放平台 HMAC 协议不同，不混用 |
| 请求记录 / 告警 | ✅ | 扩展记录 `partnerId` |

**职责**：Partner 主数据、凭证、capabilities、`client_credentials` Token 签发 / 吊销、Token introspect。  
**不负责**：开放平台业务、实例状态机、SVMP 调用。

### 3.1 Partner 管理接口（🆕 · 内部 / 运营）

| 方法 | 路径 | 说明 | 阶段 |

| 方法 | 路径 | 说明 | 阶段 |

### 3.3 Token 接口

| 方法 | 路径 | 调用方 | 说明 | 阶段 |
|------|------|--------|------|------|
| POST | `/oauth/token` | Partner（公网，经 morningglory） | 标准 OAuth2 `client_credentials` | P0 |
| POST | `/internal/partners/token` | Partner（备选） | 简化 JSON 请求体，见 [Partner 鉴权 §4.1](./开放平台Partner鉴权与隔离-落地方案.md) | P0 |
| POST | `/internal/token/introspect` | **morningglory** | 入参 token，返回 Partner 上下文 JSON | P0 |
| POST | `/internal/token/revoke` | 管理后台 | 吊销单个 token | P2 |

**Token 响应字段（对外）**

| 字段 | 说明 |
|------|------|
| `accessToken` / `access_token` | Bearer Token |
| `tokenType` | `Bearer` |
| `expiresIn` | 秒 |
| `partnerId` | 接入方 ID |

**Redis 缓存（clover 写入）**

| 键 | 值 |
|----|-----|
| `partner:token:{sha256(accessToken)}` | `{ subjectType, partnerId, capabilities, clientId, expiresAt }` |
| `partner:credential:{clientId}` | 凭证元数据（不含明文 secret） |

> 与用户 Token 分离：用户 `ACCESS_TOKEN:*`；Partner `partner:token:*`。`subjectType` 固定 `PARTNER`。

## 4. open-api-service · 开放平台业务

**职责**：实现 `/api/open/v1/*`、数据隔离、幂等、Webhook 出站、SVMP 适配。  
**不负责**：Partner Token 签发、门户 IAM、部侧加解密（属 Regulatory 模块）。

**服务内上下文**：从请求头 `X-Partner-Id` 读取；**禁止**使用 body/query 中的 partnerId 作鉴权。

### 4.1 对外开放 REST（📄 契约 · 🆕 实现）

### 4.3 Webhook 出站（open-api-service → Partner）

Partner **实现接收端**；平台主动 `POST` Partner 的 `callbackUrl`。

| 项 | 说明 |
|----|------|
| 触发模块 | open-api-service 事件投递器 |
| 签名头 | `X-Webhook-Signature`（HMAC-SHA256，密钥 `webhook_secret`） |
| 配置来源 | Feign clover 或本地缓存 Partner 表 |
| 成功响应 | HTTP 200，`{ "received": true }` |
| 失败 | 平台异步重试；业务码 `50002` |

**事件类型与 capability**

| eventType | 需 capability | 典型 payload 字段 | 阶段 |
|-----------|---------------|-------------------|------|
| `TASK_COMPLETED` | EVENT_SUBSCRIBE | taskId, extTaskId, status | P1 |
| `TASK_FAILED` | EVENT_SUBSCRIBE | taskId, reason | P1 |
| `INSTANCE_STATUS_CHANGED` | EVENT_SUBSCRIBE | vulInfoID, vulInfoStat | P1 |
| `INSTANCE_REMEDIATED` | EVENT_SUBSCRIBE | vulInfoID, vulInfoStat=5 | P1 |
| `INSTANCE_ARCHIVED` | EVENT_SUBSCRIBE | vulInfoID, vulInfoStat=9, lvRsn | P1 |
| `INSTANCE_VERIFY_FIX_COMPLETED` | EVENT_SUBSCRIBE | vulInfoID, vulInfoStat=6/7/10 | P1 |
| `EXPORT_READY` | EVENT_SUBSCRIBE | exportId, downloadUrl, exportStage | P2 |

`exportStage`：`TASK_COMPLETED` / `VERIFY_SCAN` / `VERIFY_FIX_SCAN`。

## 5. SVMP 执行层 · 引擎接口（内部）

**职责**：扫描、漏洞处置、修复核验执行。  
**调用方**：仅 **open-api-service**（`SvmpEngineAdapter`）。Partner **不直连**。

### 5.1 引擎 REST 对照（平台内部）

| 开放平台动作 | SVMP 接口 | 入参要点 |
|--------------|-----------|----------|
| 创建任务 | `POST /task/create` | 扫描目标、模板；**不带** extTaskId |
| 创建前选设备 | `GET /device/list` | 按目标解析设备 |
| 查询进度 | `GET /task/progress` | **仅** SVMP taskId |
| 实例列表 | `POST /vuln/disposal/list` | SVMP taskId + 筛选 |
| 实例详情 | `GET /vuln/disposal/detail` | vulnDisposalId |
| 验证 / 修复 / 备案 | `POST /vuln/disposal/disposal` | vulnDisposalId + 处置参数 |
| 修复核验（触发引擎） | `POST /vuln/disposal/verify` | vulnDisposalId + 核验参数 |
| 修复核验（回写） | — | 平台状态机直接写 6/7/10，不调 SVMP |

### 5.2 ID 映射（open-api-service 维护）

| 层 | 字段 | 说明 |
|----|------|------|
| Partner ↔ 平台 | `extTaskId` | 仅创建任务 |
| Partner ↔ 平台 | `taskId` | 平台对外任务 ID |
| 平台 ↔ SVMP | `engineTaskId` | SVMP 返回的 taskId |
| Partner ↔ 平台 | `vulInfoID` | 实例业务主键 |
| 平台 ↔ SVMP | `vulnDisposalId` | 由 vulInfoID 查映射 |

### 5.3 SVMP 模块与服务（✅ 现有 · 参考）

| 模块 | 路径 | 说明 |
|------|------|------|
| vul-pass | `project_backend/svmp/vul-pass` | 扫描任务、子任务、结果 |

## 6. Regulatory 模块 · 部侧合规（内部）

**职责**：工信部 2025 接口 3 入站、接口 4 回传；与开放平台 **并行**，不替代 `/api/open/v1`。  
**实现位置**：建议独立 `regulatory-adapter` 或 open-api-service 子模块（设计方案 §七）。

| 接口 | 方向 | 说明 | 阶段 |
|------|------|------|------|
| `RegulatoryIngress.receiveOrder` | 部侧 → 平台 | 接口 3 入站；SM2RSA + SM4-GCM 解密 | P0（合规） |
| `RegulatoryEgress.reportOrder` | 平台 → 部侧 | 接口 4 回传；`vulInfoLst` 组装 | P0（合规） |
| WorkOrder 编排 | 内部 | orderSubType 31–44 映射 SVMP / Open API | P0 |

**与开放平台关系**：Partner 触发的任务若需报送部侧，须关联 `orderID` 或进入同一编排上下文（设计方案 §7.6）。

| 42901 | 限流 | morningglory |
| 40001–40005 | 参数 / 状态机 / 资源 | open-api-service |

| 40901 | extTaskId 幂等冲突 | open-api-service |
| 50001 | 引擎调用失败 | open-api-service（SVMP 超时/异常） |
