License: Proprietary

对外第三方集成统一契约（Partner 接入）。

  • 接口规范：docs/external/开放平台API接口规范.md
  • 漏洞实例字段与部侧表34 同名：vulInfoID、vulInfoStat、vulName、vulNetAddr、remedDesc、srcMethod 等
  • Base path：/api/open/v1
  • 鉴权：v1.0.x 仅支持 Authorization: Bearer（OAuth 2.0 Client Credentials）；API Key + HMAC 暂未开放
  • 响应包装：除特别说明外，HTTP 200 仍使用业务 code 区分成功/失败
  • 幂等：创建任务用 extTaskId 或 Idempotency-Key；实例写 {动作}:{vulInfoID}:{clientRequestId}；批量 {动作}:batch:{clientBatchId}（详见接口规范 §4.2，默认保留 24h）
  • 调 SVMP：extTaskId 不下发；查询进度仅 SVMP taskId

生命周期

排查 → 验证 → 处置（POST .../remediate，含已修复 5 与修复失败 9）→ 修复核验（verify-fix）

tasks

扫描/排查任务

创建扫描任务（XML 配置）

需能力 TASK_WRITE。见接口规范 §5.1.1：提交 type（1/2/3）+ file XML。 file 根元素 <scanTask> 须含 <server>（taskName / priority / targets / 可选 callbackUrl）及根级 <targets>（登陆检查，可为空）。 可引用平台模板 ID，或内联扫描阶段于 <server> + 根级 <report>，见附录 G。 模板 ID 缺省 0 表示按 type 自动匹配。extTaskId 为 Partner 幂等键。

string <= 128 characters

处置（含已修复与修复失败/备案）

需能力 INSTANCE_REMEDIATE。 前置：vulInfoStat ∈ {2, 7}，且 ≠ 3。 可修复类 srcMethod（附录 D，1050–1053）→ vulInfoStat=5；修复失败/备案 → vulInfoStat=9，须传 lvRsn（附录 E）、archiveReason。 工单字段（必填）：srcTktRole/dstTktRole、assignerDept/handlerDept；派单人/处置人邮箱与电话各至少填一项。
